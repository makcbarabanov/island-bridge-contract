## Алгоритм деплоя (строго по шагам)

1. Перейти в каталог:
   - `/home/makc/Apps/island`
2. Подтянуть код:
   - `git pull origin main --ff-only`
3. Если в `_sql/` есть новые миграции, сначала сделать бэкап БД:
   - `bash db_backups/bu_db.sh`
   - или штатный скрипт бэкапа, если предусмотрен окружением.
4. Перезапуск только через Docker Compose:
   - `docker compose down`
   - `docker compose up -d --build`
5. Применение миграций внутри контейнера:
   - `docker compose exec app python3 run_migrate.py _sql/имя_файла.sql`
6. Проверка:
   - `docker ps`
   - `docker compose logs --tail=50`

## Политика БД

- Миграции строго additive: только добавляют/расширяют, ничего не удаляют.

## Критичность нарушений

- Любой рестарт мимо `docker compose` расценивается как критическое нарушение инфраструктуры.
